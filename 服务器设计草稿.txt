Linux WSL1 Ubuntu20
C++ 17 GUN
点播
线程池 + I/O多路复用 (参考 TinyWebServer)
RTSP (控制 TCP)
视频
RTP   (数据 UDP)
RTCP (客户端周期反馈传输质量 UDP)
音频
RTP   (数据 UDP)
RTCP (客户端周期反馈传输质量 UDP)

应用层：RTSP协议解析
    ↓
业务层：会话管理、状态机
    ↓
网络层：ET模式 + 非阻塞IO + 缓冲区 + reactor模式(线程池子线程读取缓冲区数据)
    ↓
传输层：TCP/UDP (RTP/RTCP)

模块
1, 线程池
2, 日志(spdlog)
3, 定时器
4, 配置管理Json解析(jsoncpp)
5, 读取H264文件并分帧
6, 网络I/O,线程池 + I/O多路复用
7, 会话管理,维护每个用户连接的状态
8, RTSP协议解析(需要扩展jwt-token认证,URL参数传递Token)
	(没有认证服务器,如果需要开启认证需要一个认证服务器)
9, RTP封装发送
10, 路由器(url路由到对应资源)
11, 解析命令行参数

rtsp://192.168.1.100:8554/video?token=eyJhbGciOiJIkpXVCJ9...

命令行参数

参数	类型	默认值	说明
--rtsp-port	int	8554	RTSP服务监听端口
--rtsp-host	string	0.0.0.0	监听地址（0.0.0.0表示所有网卡）
--rtp-port-range	string	10000-20000	RTP/RTCP端口池范围
--max-connections	int	1000	最大并发连接数

--auth-enabled	bool	false	是否开启认证
--auth-type	string	jwt	认证类型：jwt/basic/digest/none
--jwt-secret	string	-	JWT密钥（必须配置）
--jwt-expire	int	3600	Token有效期（秒）
--jwt-refresh-enabled	bool	true	是否支持Token刷新
--auth-api-port	int	8080	HTTP认证API端口
--auth-api-enabled	bool	true	是否启用HTTP认证接口

--log-level	string	info	日志级别：trace/debug/info/warn/error/fatal/off
--log-dir	string	./logs	日志目录
--log-file	string	rtsp-server.log	日志文件名
--log-max-size	int	100	单个日志文件最大大小（MB）
--log-max-files	int	10	保留的日志文件数量
--log-to-console	bool	true	是否输出到控制台

--media-dir	string	./media	媒体文件目录
--video-dir	string	./media/videos	视频文件目录
--audio-dir	string	./media/audios	音频文件目录

--thread-pool-size	int	8	线程池线程数
--worker-threads	int	4	工作线程数
--session-timeout	int	60	会话超时时间（秒）
--buffer-size	int	8192	UDP缓冲区大小（字节）
--tcp-keepalive	bool	true	是否开启TCP KeepAlive
--tcp-keepalive-time	int	60	TCP保活时间（秒）

--config-file string "config.json" 读取配置文件(代替命令行参数)
