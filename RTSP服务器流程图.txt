┌─────────────────────────────────────────────────────────────────────────────┐
│                          关键流程说明                                          │
└─────────────────────────────────────────────────────────────────────────────┘

1. 初始化流程
   - 创建 Server 对象
   - 加载配置文件（端口、线程数、日志等）
   - 初始化 SessionManager（会话管理器）
   - 初始化 PortPool（端口池，范围 10000-20000）
   - 创建 ThreadPool（线程池，默认4个线程）
   - 创建 EpollManager（事件管理器）
   - SocketManager 创建并绑定监听端口 554
   - 设置定时器（每60秒清理超时会话）

2. 事件循环流程
   - epoll_wait() 阻塞等待事件
   - 事件触发后，根据 fd 类型分发处理：
     * rtsp_fd：新连接 → handleNewConnection()
     * client_fd：客户端请求 → handleClientRequest()
     * timer_fd：定时器 → handleTimerEvent()

3. 新连接处理
   - accept() 接受连接，获取 client_fd
   - 创建 Connection 对象（存储客户端信息）
   - 设置非阻塞模式
   - 添加到 epoll 监听
   - 注册事件处理器

4. 客户端请求处理
   - read_data() 读取 RTSP 请求
   - 解析请求类型（DESCRIBE、SETUP、PLAY 等）
   - 根据请求类型执行业务逻辑
   - 构建响应并发送回客户端

5. 定时器处理
   - 每60秒触发一次
   - 调用 SessionManager::CleanupTimeoutSessions()
   - 删除超过300秒未活动的会话
   - 释放相关 UDP 端口


┌─────────────────────────────────────────────────────────────────────────────┐
│                          RTSP 协议端口分配                                     │
└─────────────────────────────────────────────────────────────────────────────┘

TCP 端口（控制通道）:
  - 554: RTSP 标准端口，监听所有客户端连接
  - 每个客户端: 1 个 TCP fd，用于 RTSP 命令交互

UDP 端口（数据通道）:
  - 视频流:
    * 视频 RTP: 动态分配（如 10000）
    * 视频 RTCP: 动态分配（如 10001）
  - 音频流:
    * 音频 RTP: 动态分配（如 10002）
    * 音频 RTCP: 动态分配（如 10003）

端口分配范围:
  - UDP 端口池: 10000 - 20000
  - 每个 Session 最多占用 4 个 UDP 端口


┌─────────────────────────────────────────────────────────────────────────────┐
│                          架构模式说明                                          │
└─────────────────────────────────────────────────────────────────────────────┘

1. Reactor 模式（事件驱动）
   - 使用 epoll 实现多路复用
   - 单线程事件循环 + 事件处理器
   - 高效处理大量并发连接

2. 线程池模式
   - 固定数量的工作线程
   - 任务队列 + 条件变量同步
   - 适合 CPU 密集型任务（如视频编码）

3. 对象池模式（端口池）
   - PortPool 管理可用的 UDP 端口
   - 位图实现，快速分配/释放
   - 线程安全，支持并发访问

4. RAII 模式
   - 智能指针管理 Connection 生命周期
   - 析构函数自动释放资源
   - 防止内存泄漏
